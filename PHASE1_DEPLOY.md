## 📋 Шаг 1: Настройка Supabase

### 1.1 Применить миграцию базы данных

1. Откройте ваш проект Supabase: https://supabase.com/dashboard
2. Перейдите в **SQL Editor**
3. Откройте файл `supabase/phase1-migration.sql`
4. Скопируйте весь SQL код и вставьте в редактор
5. Нажмите **Run** для выполнения миграции

Миграция добавит:
- Новые поля в таблицу `products` (brand, color, size, barcode, is_active)
- Новые поля в таблицу `categories` (parent_id, sort_order)
- Таблицу `product_imports` для истории импорта
- Таблицы `inventory_sessions` и `inventory_items` для инвентаризации
- Функцию `apply_inventory_session` для применения результатов
- Все необходимые индексы и RLS политики

### 1.2 Проверка миграции

Выполните следующий SQL для проверки:

```sql
-- Проверка новых полей в products
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'products' 
AND column_name IN ('brand', 'color', 'size', 'barcode', 'is_active');

-- Проверка новых таблиц
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('product_imports', 'inventory_sessions', 'inventory_items');
```

Должны вернуться все поля и таблицы.

1. Перейдите на https://vercel.com
2. Нажмите **Add New** → **Project**
3. Импортируйте ваш GitHub репозиторий
4. Настройте переменные окружения:
   - `NEXT_PUBLIC_SUPABASE_URL` = `https://rbvoabyddxjjhekrrafa.supabase.co`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = `ваш_anon_key`
5. Нажмите **Deploy**

**Вариант 2: Через Vercel CLI**

```bash

# Логин

# Деплой
vercel

# Добавление переменных окружения
vercel env add NEXT_PUBLIC_SUPABASE_URL
```

## 🔐 Шаг 5: Настройка безопасности

### 5.1 Настройка Supabase Auth

1. В Supabase перейдите в **Authentication** → **URL Configuration**
2. Добавьте URL вашего Vercel приложения в **Site URL**
3. Добавьте URL в **Redirect URLs**:
   - `https://ваш-домен.vercel.app/auth/callback`
   - `https://ваш-домен.vercel.app/dashboard`

### 5.2 Проверка RLS политик

Все таблицы защищены Row Level Security. Проверьте в Supabase:

```sql
-- Проверка RLS для новых таблиц
SELECT tablename, rowsecurity 
FROM pg_tables 
WHERE schemaname = 'public' 
AND tablename IN ('product_imports', 'inventory_sessions', 'inventory_items');
```

Все должны иметь `rowsecurity = true`.

## 🔧 Устранение неполадок

### Проблема: Ошибка при импорте товаров

**Решение:**
- Проверьте формат Excel файла (должен быть .xlsx)
- Убедитесь, что обязательные поля (SKU, название) заполнены
- Проверьте, что категории существуют в системе

### Проблема: Не применяются результаты инвентаризации

**Решение:**
- Убедитесь, что функция `apply_inventory_session` создана в Supabase
- Проверьте, что все товары имеют фактическое количество
- Проверьте статус сессии (должен быть "in_progress")

### Проблема: Ошибка при переводе товаров

**Решение:**
- Проверьте, что товары есть в локации "Откуда"
- Убедитесь, что количество не превышает доступное
- Проверьте, что обе локации существуют

## 📝 Новые возможности

### Массовая загрузка товаров

**Формат Excel файла:**
- `sku` - артикул (обязательно)
- `name` - название (обязательно)
- `description` - описание
- `category` - название категории
- `unit` - единица измерения (по умолчанию "шт")
- `purchase_price` - цена закупки
- `sale_price` - цена продажи
- `currency` - валюта (UZS/USD)
- `brand` - бренд
- `color` - цвет
- `size` - размер
- `barcode` - штрих-код
- `min_stock_level` - минимальный остаток

### Инвентаризация

**Процесс:**
1. Создание сессии инвентаризации для локации
2. Система загружает текущие остатки
3. Ввод фактических остатков вручную
4. Автоматический расчет расхождений
5. Применение результатов (обновление остатков)

**Статусы:**
- `draft` - черновик
- `in_progress` - в процессе
- `completed` - завершена
- `cancelled` - отменена

# Быстрый старт 🚀

## За 5 минут до запуска

### 1. Установка (1 минута)
```bash
cd warehouse-saas
npm install
```

### 2. Настройка Supabase (2 минуты)

1. Создайте проект на [supabase.com](https://supabase.com)
2. В SQL Editor выполните скрипт из `supabase/schema.sql`
3. Скопируйте URL и API ключ из Settings → API

### 3. Переменные окружения (30 секунд)

Откройте `.env.local` и замените:
```env
NEXT_PUBLIC_SUPABASE_URL=https://ваш-проект.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_ключ_здесь
```

### 4. Запуск (30 секунд)
```bash
npm run dev
```

Откройте http://localhost:3000

### 5. Первые шаги (1 минута)

1. Зарегистрируйтесь на `/register`
2. Войдите в систему
3. Готово! 🎉

## Быстрая настройка данных

### Добавьте первую локацию
1. Перейдите в "Локации"
2. Нажмите "Добавить локацию"
3. Заполните:
   - Название: "Главный склад"
   - Тип: "Склад"
   - Адрес: ваш адрес
4. Сохраните

### Добавьте первый товар
1. Перейдите в "Товары"
2. Нажмите "Добавить товар"
3. Заполните:
   - Артикул: "SKU-001"
   - Название: "Тестовый товар"
   - Единица: "шт"
   - Цена закупки: 100
   - Цена продажи: 150
4. Сохраните

### Создайте приход товара
1. Перейдите в "Транзакции"
2. Нажмите "Новая транзакция"
3. Выберите:
   - Тип: "Приход товара"
   - Локация: "Главный склад"
4. Добавьте товар:
   - Товар: "Тестовый товар"
   - Количество: 100
   - Цена: 100
5. Сохраните

### Проверьте остатки
1. Перейдите в "Остатки"
2. Увидите: Тестовый товар - 100 шт на Главном складе

## Проблемы?

### Ошибка подключения к Supabase
- Проверьте URL и ключ в `.env.local`
- Перезапустите `npm run dev`

### Ошибка "relation does not exist"
- Выполните SQL скрипт из `supabase/schema.sql`

### Не работает регистрация
- Проверьте, что Email провайдер включен в Supabase
- Authentication → Providers → Email → ON

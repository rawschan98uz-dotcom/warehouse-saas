# Система управления складом (Warehouse Management SaaS)

Современная SaaS система для управления складом, товарами и остатками с поддержкой мультитенантности.

## Возможности

### Основные функции
- ✅ **Аутентификация** - Регистрация и вход через Supabase Auth
- ✅ **Управление товарами** - Полный CRUD для каталога товаров
- ✅ **Управление локациями** - Склады и магазины
- ✅ **Учет остатков** - Отслеживание товаров по локациям
- ✅ **Приходы товаров** - Оприходование товаров на склад
- ✅ **Продажи** - Продажи со склада и магазина
- ✅ **Переводы** - Перемещение товаров между локациями
- ✅ **Расходы** - Списание и расходы товаров
- ✅ **Поставщики** - Управление базой поставщиков
- ✅ **Отчеты** - Аналитика и статистика
- ✅ **Уведомления** - Оповещения о низких остатках

### Технологии
- **Frontend**: Next.js 14+ (App Router), React, TypeScript
- **UI**: Shadcn/ui + Tailwind CSS
- **Backend**: Supabase (PostgreSQL + Auth)
- **Deployment**: Vercel

## Установка и настройка

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com)
2. Перейдите в SQL Editor и выполните скрипт из файла `supabase/schema.sql`
3. Скопируйте URL проекта и Anon Key

### 3. Настройка переменных окружения

Откройте файл `.env.local` и заполните:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 4. Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## Использование

### Первый запуск

1. Зарегистрируйтесь на странице `/register`
2. Войдите в систему
3. Добавьте локации (склады/магазины)
4. Добавьте товары в каталог
5. Создайте приход товаров для пополнения остатков

### Основные операции

**Приход товаров:**
1. Перейдите в "Транзакции" → "Новая транзакция"
2. Выберите тип "Приход товара"
3. Выберите локацию и добавьте товары
4. Сохраните транзакцию

**Продажа:**
1. Создайте новую транзакцию типа "Продажа"
2. Выберите локацию (склад или магазин)
3. Добавьте товары и количество
4. Система автоматически спишет товары с остатков

**Перевод между локациями:**
1. Создайте транзакцию типа "Перевод"
2. Выберите локацию "Откуда" и "Куда"
3. Добавьте товары для перемещения
4. Остатки обновятся автоматически

## База данных

### Основные таблицы

- `organizations` - Организации (мультитенантность)
- `profiles` - Профили пользователей
- `organization_users` - Связь пользователей с организациями
- `products` - Каталог товаров
- `locations` - Склады и магазины
- `inventory` - Остатки товаров по локациям
- `transactions` - Все операции (приходы, продажи, переводы)
- `transaction_items` - Позиции транзакций
- `suppliers` - Поставщики

### Row Level Security (RLS)

Все таблицы защищены политиками RLS. Пользователи видят только данные своей организации.

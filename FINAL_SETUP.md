## ⚠️ Осталось только одно - выполнить SQL скрипт!

### Шаг 1: Откройте Supabase SQL Editor

1. Перейдите на https://supabase.com/dashboard/project/rteauhmgpyfkianlvnbh
2. В левом меню выберите **SQL Editor**
3. Нажмите **New query**

### Шаг 2: Выполните SQL скрипт

1. Откройте файл `supabase/reset-and-setup.sql` из проекта
2. Скопируйте **ВЕСЬ** SQL код (Ctrl+A, Ctrl+C)
3. Вставьте в SQL Editor в Supabase (Ctrl+V)
4. Нажмите **Run** (или Ctrl+Enter)
5. Подождите 5-10 секунд

### Шаг 3: Проверьте результат

Должно появиться сообщение: **"Success. No rows returned"**

Это означает, что:
- ✅ Все старые таблицы удалены
- ✅ Созданы 10 новых таблиц
- ✅ Настроены индексы
- ✅ Включен Row Level Security
- ✅ Созданы политики доступа
- ✅ Настроены триггеры

### Шаг 4: Проверьте таблицы

1. В левом меню выберите **Table Editor**
2. Убедитесь, что созданы таблицы:
   - ✅ organizations
   - ✅ profiles
   - ✅ organization_users
   - ✅ categories
   - ✅ locations
   - ✅ suppliers
   - ✅ products
   - ✅ inventory
   - ✅ transactions
   - ✅ transaction_items

## 🚀 После выполнения SQL:

### Откройте приложение

1. Перейдите на http://localhost:3000
2. Нажмите "Зарегистрироваться"
3. Создайте аккаунт:
   - Email: ваш email
   - Пароль: минимум 6 символов
   - Имя: ваше имя
4. Войдите в систему

### Начните работу!

После входа вы увидите дашборд. Теперь можно:

1. **Добавить локацию** (склад или магазин)
   - Перейдите в "Локации"
   - Нажмите "Добавить локацию"
   - Заполните данные

2. **Добавить товар**
   - Перейдите в "Товары"
   - Нажмите "Добавить товар"
   - Заполните данные

3. **Создать приход товара**
   - Перейдите в "Транзакции"
   - Нажмите "Новая транзакция"
   - Выберите тип "Приход товара"
   - Добавьте товары

4. **Проверить остатки**
   - Перейдите в "Остатки"
   - Увидите все товары по локациям

## 🆘 Если возникли проблемы:

### Ошибка при выполнении SQL
- Убедитесь, что скопировали ВЕСЬ код из файла
- Попробуйте выполнить еще раз

### Ошибка "relation already exists"
- Это нормально, скрипт сначала удаляет старые таблицы
- Просто выполните скрипт еще раз

### Не можете зарегистрироваться
- Проверьте, что SQL скрипт выполнен успешно
- Проверьте в Table Editor, что таблицы созданы
